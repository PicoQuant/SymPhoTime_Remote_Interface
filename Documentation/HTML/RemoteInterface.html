Scripted Class

RemoteInterface

  [Related Topics]

Synopsis:

Interface to remote control the SymPhoTime 64 data acquisition via TCP/IP

Notice: In SymPhoTime 64, there will never more than one instance of the RemoteInterface object be active.

Initially this will be the first instance created: All instances created subsequently will be set to inactive state. With the TryActivate method, you may try to swich an inactive instance to be the currently active one. This works only if there is currently no active TCP/IP connection to a client running. Check the field "IsActive" to find out, whether the method was successfully setting the current instance to be the active one or not.

Declaration Example:

SomeRemoteInterface: RemoteInterface;

RI_RegResult: Int		 Method to register the variable given in <RI_Variable> as parameter to pass back to the calling client.
Registered parameters may be grouped into contexts, identified by the context mask given in <RI_Context>, e.g. rfsOnStart or rfsConsec as to name some predefined context masks. Thus the same variable might be registered for more than one context. Passing either -1 or the predefined constant rfsAlways as context mask identifies the parameter as "always to be sent".
UnregisterRIVars Context: Int Method to clear (i.e. unregister) previously registered parameters.
You may restrict the clear to the contexts identified by the bits set in the context mask; Passing -1 for the context, all parameters of all groups are unregistered.
SendRIVars Context: Int Method to send previously registered feedback parameters to the client.
You may restrict the parameters to be sent by the contexts identified with the bits set in the context mask; Passing either -1 or the predefined constant rfsAlways as context mask, all parameters of all groups are sent. Their position and order in the messages to be passed to the client is arbitrary.
